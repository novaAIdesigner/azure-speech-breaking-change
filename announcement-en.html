Important: Action required before July 1, 2026

Azure AI Speech SDK: Certificate Revocation List (CRL) compatibility update

Summary: On Linux and Android, Speech SDK versions earlier than 1.48.2 can fail to connect to Azure Speech Service due to CRL partitioning changes in the certificate ecosystem. If you use cloud speech features, you should upgrade or mitigate before July 1, 2026.

Affected Platforms

Linux, Android

Affected Versions

Speech SDK < 1.48.2

Not Affected

Windows, iOS, macOS, SDK 1.48.2+

Quick Impact Check

  • You run Speech SDK on Linux or Android.
  • You use SDK version earlier than 1.48.2.
  • Your app connects to Azure Speech cloud endpoints (STT/TTS).

If all are true, action is required.

Deployment Impact Summary

Deployment Type Impact Action
Embedded only (no cloud) Not impacted No action needed
Hybrid (cloud + embedded fallback) Cloud STT/TTS may fail Action required for cloud path
Cloud only Cloud STT/TTS may fail with no fallback Action required

Required Action (Before July 1, 2026)

Option 1 (Recommended): Upgrade to SDK 1.48.2+

  • Includes fix for partitioned CRL handling.
  • Validate by confirming no error 44 in logs.
  • Test cross-region connections if applicable.

Option 2: Disable CRL checking

  • Set OPENSSL_DISABLE_CRL_CHECK=true.
  • Use when immediate upgrade is not possible.
  • Still plan upgrade as the long-term fix.
// C# example
config.SetProperty("OPENSSL_DISABLE_CRL_CHECK", "true");

If You Cannot Fix Immediately

Temporary workaround: clear or avoid CRL disk cache by unsetting TMPDIR/TMP or deleting cached .crl files in the temp directory, then restart the app.

This is temporary only. Upgrade or disable CRL checking is still required.

How to Detect the Issue

  • Connection failures with WS_OPEN_ERROR_UNDERLYING_IO_OPEN_FAILED.
  • TLS log includes error 44: X509_V_ERR_DIFFERENT_CRL_SCOPE.
  • Often appears after certificate rotation or cross-region usage.

Risk If No Action Is Taken

  • Unplanned cloud STT/TTS outages on Linux/Android.
  • No exact warning window (depends on cert rotation timing).
  • Service remains unstable until upgrade/mitigation is applied.

Timeline

  1. Now: Confirm platform, version, and deployment type.
  2. February 2026: SDK 1.48.2 released with fix.
  3. July 1, 2026: Deadline to upgrade or mitigate.
  4. After deadline: Unpatched apps may intermittently fail as certs rotate.
