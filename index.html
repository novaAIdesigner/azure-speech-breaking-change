Azure Speech SDK（Linux/Android）CRL 兼容性风险：2026/7/1 前必须行动

行业证书体系演进(SC-089)触发，Speech SDK强制升级需求

为什么会发生（根因）

  • 📜证书机构正在转向新的 CRL 架构；旧版 SDK（< 1.48.2）在 Linux/Android 上 CRL 缓存处理不兼容。
  • 🛡️可能触发 OpenSSL 验证错误并导致连接失败。
  • ⚡表现：证书校验失败、OpenSSL
    error 44 / X509_V_ERR_DIFFERENT
    _CRL_SCOPE。
  • ✅Windows / iOS / macOS 不受影响；聚焦 Linux / Android。

谁会受影响

🚗📱☁️ 平台

Linux Android 全语言绑定

场景

Hybrid（Cloud + Embedded） Cloud Only 纯 Embedded 且从不连云端
通常不受影响

关键触发条件

启用 certificate caching（Cert/CRL 缓存）时风险最高。

自检与缓解

行动清单

1) 立即自查

平台 / 版本 / 是否连云 / 是否启用缓存。

2) 优先缓解

优先远程配置禁用 CRL 检查；可重启作为临时手段。

3) 最终修复

升级到 Speech SDK v1.48.2+（长期正确解）。
